ESPHome Device Builder blueos-relay

Factory project for the Waveshare ESP32-S3-Relay-6CH + Pico-RTC-DS3231, pre-wired for BlueOS MQTT.

1. MQTT broker (from BlueOS Beacon)

Reads the live BlueOS mDNS hostname via Beacon (GET /v1.0/hostname) instead of a hardcoded blueos.local. Rename your vehicle in BlueOS and re-run this step — no reflash of firmware needed.

Static IP fallback (e.g. 192.168.1.113) is useful if mDNS is unreliable on the site Wi‑Fi.

2. Wi‑Fi credentials

Same network as the BlueOS host / Pi. Written once into secrets.yaml on the extension's persistent volume.

3. API / OTA secrets

Random per-device secrets used by ESPHome's native API and OTA updates. Generated automatically the first time you save — regenerate only if you suspect they leaked.

4. Flash the ESP32

Checking for USB serial devices…

First flash (blank board): plug the ESP32 directly into a USB port on the BlueOS Pi, then click Refresh below. USB‑JTAG can be flaky on some boards — if the port doesn't show up, unplug/replug or try a different USB cable/port.

Already flashed (e.g. running at a known IP): no USB needed — use OTA install from the dashboard by targeting the device's hostname or IP; it uses the ota_password / api_encryption_key saved above.

Topic prefix blueos/relay is shared with blueos-site-stack (Mosquitto/Influx) and blueos-site-ui (Grafana + relay control page). Config directory: /config.
